# Issue: Generate stage_*.sh scaffolding scripts

## Epic: EPIC-003 — Build Config + CI Generator

## Type: Feature

## Priority: Medium

### Description
Generate `.pi/scripts/ci/stage_*.sh` scaffolding scripts in the project's `.pi/` directory that match the validators the architecture requires. These provide the CI pipeline with executable stages for build, test, lint, format, security audit, and more.

### Acceptance Criteria
- [ ] `stage_build.sh` — runs language-appropriate build command
- [ ] `stage_test.sh` — runs language-appropriate test command
- [ ] `stage_lint.sh` — runs language-appropriate lint command
- [ ] `stage_security.sh` — runs security audit command
- [ ] `stage_integration.sh` — runs integration tests
- [ ] Each script uses `{{BUILD_COMMAND}}`, `{{TEST_COMMAND}}`, etc. from template context
- [ ] Scripts are generated only for active validators
- [ ] Scripts exit 0 on success, 1 on failure

### Implementation Notes
- Files: `.pi/scripts/ci/stage_*.sh` (generated by ci-generator.ts)
- Script content follows existing `.pi/scripts/ci/` patterns
- Template placeholders from `TemplateContext` substituted during generation

### Dependencies
- Depends on: Issue #007 (ci-generator.ts core)

### Estimated Scope
- Files: 5-7 generated scripts
- Lines: ~30-50 per script
- Scope: Moderate

### Testing Requirements
- Unit test: correct number of scripts generated for given validators
- Unit test: scripts contain correct build/test commands from context
